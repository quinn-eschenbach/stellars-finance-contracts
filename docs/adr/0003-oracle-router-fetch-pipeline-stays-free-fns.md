# OracleRouter fetch pipeline stays as free-function composition

`OracleRouter::fetch_and_validate_price` composes five free functions (`query_sources`, `insertion_sort`, `median_idx`, `deviation_bps`, `dedup_sources`) rather than living behind an `OracleQuorum` wrapper type. The architecturally cleaner alternative was a type owning the config + sources + pipeline as methods, keyed by symbol — but the pipeline has exactly one consumer (`get_price`), comprehensive test coverage targeting each failure mode (NoPriceSources, StalePrice, InsufficientSources, PriceDeviationTooHigh, DeviationOverflow, exact-boundary cases), and reads linearly top-to-bottom in 45 lines. One adapter is a hypothetical seam, not a real one: deleting `OracleQuorum` would leave the code in the same shape it has today. Revisit if a second consumer emerges — a keeper-side oracle health check that needs to fetch without emitting the `PriceFetch` event, an analytics path that wants quorum status without the median, or any caller that needs to construct a quorum from a different source list than what's in storage — at which point the wrapper concentrates the dedup + threshold + median + deviation invariants across consumers and free-fn composition becomes the brittleness.
